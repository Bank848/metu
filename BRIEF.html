Database
  • Postgres 16 (Neon)
  • Prisma 5 ORM
  • 20 entities · 3 migrations
  • Singapore region
  • Pooled + unpooled connections
Infra · Tooling
  • Vercel (auto-deploy on main)
  • npm workspaces (monorepo)

Note: Tailwind v3 + Next 14 deliberately chosen over the friend repo's Next 16 + Tailwind v4 to keep the demo deploy stable for the deadline. Schema is custom (no better-auth) — Neon prod is untouched.

Run it locally

1. Clone + install

git clone https://github.com/Bank848/metu.git
cd metu
npm install

2. Boot local Postgres

docker compose up -d
npm run db:generate
npm run db:migrate
npm run db:seed

3. Start the dev server

npm run dev -w @metu/web
# → http://localhost:3000

4. Useful URLs

  • • localhost:3000 — the app
  • • localhost:3000/api/health — sanity check
  • • localhost:8081 — Adminer to inspect DB live during demo

On Windows PowerShell, $env:DATABASE_URL = "..." to point at Neon if you'd rather seed the production DB instead of local Postgres.

Why these choices

Next.js 14 (not 16) · Tailwind v3 (not v4) · React 18 (not 19)

The friend repo runs on the new majors. We deliberately stayed one version behind because every breaking change risks a Vercel deploy regression days before the in-class demo. Bumped to 14.2.35 for security patches; held off on the major jump.

JWT in httpOnly cookie (not better-auth)

Three lines of jsonwebtoken + bcrypt covers our entire auth surface for a class demo. better-auth is a great library but adds a migration, OAuth provider config, and four more tables we don't need.

Neon (not Supabase) for production Postgres

Neon's Vercel integration auto-injects DATABASE_URL into all environments and serverless-friendly pooling means cold starts don't exhaust connections. Free tier (0.5 GB) is plenty for the demo.

Server components call Prisma directly (not /api)

Earlier the home page tried to fetch('https://VERCEL_URL/api/stats') from inside a server component, which hit Vercel's deployment-protection wall and returned 0s. Moved server-side reads into lib/server/queries.ts — direct Prisma, no HTTP roundtrip, no auth wall.

Curated Unsplash photo pools by category

The first seed used picsum.photos which returns random images per seed — so "Thai-Pop Instrumental" sometimes showed an American flag. Switched to per-category Unsplash pools with a deterministic hash so themed images stay stable across re-seeds.

Visual port without a stack rewrite

Adopted the friend repo's neo-luxury palette + glassmorphism + light-sweep + gold gradient + ProductCard pattern by porting tokens, fonts, and 3 new visual primitives — without changing Next, Tailwind, auth, or schema. The look is theirs; the engine is ours.
